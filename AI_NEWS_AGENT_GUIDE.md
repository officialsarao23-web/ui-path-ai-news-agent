# AI News Agent — Complete Guide

A UiPath Coded Agent built with LangGraph that fetches daily AI news and sends it to Telegram. Runs daily at 8 AM IST via Orchestrator schedule.

---

## Architecture

```
my-uipath-agent/
├── .env                          # UIPATH_PROJECT_ID (links to Studio Web cloud project)
├── pyproject.toml                # Python dependencies
├── uipath.json                   # UiPath project config (functions entry for LangGraph)
├── langgraph.json                # LangGraph graph mapping
├── bindings.json                 # Resource bindings for 4 Orchestrator Assets
├── entry-points.json             # Auto-generated entry point schema
├── project.uiproj                # UiPath project file
├── AI_NEWS_AGENT_GUIDE.md        # This guide
└── src/
    ├── __init__.py
    ├── main.py                   # StateGraph wiring — entry point
    └── nodes/
        ├── __init__.py
        ├── fetch_news.py         # Node 1: Tavily news search
        ├── summarize.py          # Node 2: Groq LLM summarization
        ├── format.py             # Node 3: Telegram message formatting
        └── send_telegram.py      # Node 4: Telegram Bot API send
```

### Graph Flow

```
START → fetch_news → summarize → format → send_telegram → END
```

## File Purposes

### Configuration Files

| File | Purpose |
|---|---|
| `pyproject.toml` | Python project metadata + dependencies (`uipath`, `uipath-langchain`, `langchain-groq`, `tavily-python`, `requests`) |
| `uipath.json` | UiPath project config. Uses `functions.graph` pointing to `src/main.py:graph` so the runtime knows this is a LangGraph agent. Sets `isConversational: false`. |
| `langgraph.json` | LangGraph middleware config — maps entrypoint name `"graph"` to the compiled `StateGraph` variable in `src/main.py`. |
| `bindings.json` | Declares 4 overridable Orchestrator Assets (`TAVILY_API_KEY`, `GROQ_API_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`) so they can be swapped per-environment. |
| `entry-points.json` | Auto-generated by `uip codedagent init`. Captures the graph's input/output schemas, node names, and edge topology. |
| `.env` | Stores `UIPATH_PROJECT_ID` — links this local project to the Studio Web cloud workspace. Used by `uip codedagent push`. |
| `project.uiproj` | UiPath project metadata file. |

### Source Code

#### `src/main.py` — Graph Wiring
- Defines `GraphInput` (trigger string), `GraphOutput` (status string), and `AgentState` (articles, summaries, message)
- Builds a `StateGraph` with 4 nodes connected sequentially
- Exports `graph` — the compiled `CompiledStateGraph` that the runtime executes

#### `src/nodes/fetch_news.py` — Node 1
- **Purpose:** Fetch top 5 AI news articles
- Fetches `TAVILY_API_KEY` from Orchestrator Assets via `sdk.assets.retrieve()`
- Calls Tavily Search API with query `"artificial intelligence AI news today"`, `search_depth="advanced"`, `max_results=5`
- Returns `{ articles: [{title, url, content}] }` to state

#### `src/nodes/summarize.py` — Node 2
- **Purpose:** Summarize each article in 2 plain-English sentences
- Fetches `GROQ_API_KEY` from Orchestrator Assets
- Uses `langchain_groq.ChatGroq` with model `llama-3.3-70b-versatile`
- Iterates over `state.articles`, sends each content to Groq, collects summaries
- Returns `{ summaries: [{title, summary, url}] }` to state

#### `src/nodes/format.py` — Node 3
- **Purpose:** Format summaries into a Telegram-ready message string
- Builds a rich text message with date, emoji bullets, summaries, and URLs
- Returns `{ message: "..." }` to state

#### `src/nodes/send_telegram.py` — Node 4
- **Purpose:** Send the formatted message to Telegram
- Fetches `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID` from Orchestrator Assets
- Calls `https://api.telegram.org/bot{token}/sendMessage` via `requests.post`
- Returns `{ status: "success" }` on success, raises `RuntimeError` on failure

## Prerequisites & Setup

### 1. Orchestrator Assets (REQUIRED before first run)

Create these 4 **String** assets in Orchestrator → Assets (same folder as the process):

| Asset Name | Example Value |
|---|---|
| `TAVILY_API_KEY` | `tvly-xxxxxxxxxxxxxxxx` |
| `GROQ_API_KEY` | `gsk_xxxxxxxxxxxxxxxx` |
| `TELEGRAM_BOT_TOKEN` | `123456:ABC-DEFxxxxxxxx` |
| `TELEGRAM_CHAT_ID` | `123456789` |

Values are raw strings — **no quotes** needed.

### 2. CLI Commands Reference

```bash
# Activate virtual environment
.venv\Scripts\activate

# Install dependencies
pip install uipath-langchain langchain-groq tavily-python requests

# Setup coded agent runtime
uip codedagent setup --force

# Regenerate entry-points.json after code changes
uip codedagent init

# Push local changes to Studio Web cloud project
uip codedagent push

# Pack into .nupkg
uip codedagent pack

# Publish to personal workspace
uip codedagent publish --my-workspace

# Publish to tenant feed
uip codedagent publish --tenant

# Run locally (test)
uip codedagent run graph '{"trigger": "daily"}'
```

## Orchestrator Schedule Setup

1. Navigate to **Orchestrator** → **Processes** → select `my-uipath-agent`
2. Go to **Triggers** tab → **Add Trigger**
3. Configure:
   - **Type:** Schedule
   - **Timezone:** `(UTC+05:30) India Standard Time`
   - **Frequency:** Daily
   - **Time:** `08:00`
   - **Process:** `my-uipath-agent`
   - **Alert:** Check "Generate alert if job started and has not completed" (timeout: 15 min)
4. Click **Create**

## How It Runs

```
8:00 AM IST
    ↓
Trigger fires → Orchestrator starts a Job
    ↓
Job runs my-uipath-agent (the LangGraph graph)
    ↓
fetch_news → Tavily API → 5 AI news articles
    ↓
summarize → Groq llama-3.3-70b → 2-sentence summaries
    ↓
format → Build Telegram message with emojis + links
    ↓
send_telegram → Telegram Bot API → Message in your chat
    ↓
Job completes → Status: Success
```
